## Blockers / Challenges Faced

| # | Issue | Root Cause | Resolution |
|---|-------|------------|------------|
| 1 | **Corrupted `infra/compute/index.ts`** | Bad merge commit duplicated imports and nested blocks, producing a 2168-line malformed file | Restored clean 1204-line version from `HEAD~1` (`4fe85f9`) |
| 2 | **S3 state bucket missing** | `pulumi stack ls` failed with `NoSuchBucket` | Created `chatbot-pulumi-state` via `aws s3api` |
| 3 | **Missing `AWS_PROFILE`** | Pulumi backend S3 access failed with credential errors | Exported `AWS_PROFILE=STX-CLOUD-PLATFORM` on all Pulumi commands |
| 4 | **BuildKit cache import failure** | `cacheFrom` referenced ECR tags that didn't exist yet (`:latest`), causing overlayfs cache corruption | Removed `cacheFrom` lines from both `awsx.ecr.Image` resources |
| 5 | **Host `node_modules` symlinks conflicting with container** | BuildKit COPY tried to replace container directories with host symlinks | Created root `.dockerignore` excluding all `node_modules` |
| 6 | **Bun workspace "infra" not found** | Root `package.json` workspaces include `"infra"`, but Dockerfile didn't copy it | Added `COPY infra/package.json infra/` |
| 7 | **Prisma generate during `prepare` script** | `bun install` triggers `prepare` which runs `prisma generate`, but `prisma/` wasn't copied yet | Moved `COPY prisma/ prisma/` before `bun install` |
| 8 | **`tsc` cannot read `tsconfig.base.json`** | `apps/workers/tsconfig.json` extends `../../tsconfig.base.json`, but file wasn't copied | Copied `tsconfig.base.json` and `nx.json` into container |
| 9 | **`next build` env validation failure** | `next.config.ts` imports `lib/env.ts` which uses `@t3-oss/env-nextjs` to validate `NEXTAUTH_SECRET` at import time | Added `ENV NEXTAUTH_SECRET=dummy-build-secret` to web-ui builder stage |
| 10 | **Monorepo path resolution inside container** | Dockerfiles flattened app into `/app/` but `tsconfig.base.json` uses `baseUrl: "."` with paths like `"libs/shared/src/index.ts"` | Rewrote both Dockerfiles to preserve full monorepo structure under `/app/` |

## Workers-specific Discovery
- `apps/workers/tsconfig.json` has `"noEmit": true`, so `tsc` emits nothing. The Nx `project.json` uses `@nx/js:tsc` which overrides this. Fixed by running `bunx tsc --noEmit false` in the Dockerfile.
